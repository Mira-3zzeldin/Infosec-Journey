# Security Policy

## Scope

This repository contains **educational notes, lab documentation, and learning resources** related to offensive security and penetration testing. It does not contain production code, live systems, or deployable software.

---

## Reporting a Vulnerability

If you discover a security concern related to this repository — such as :

- An accidentally exposed credential, API key, or sensitive personal data
- A script or tool that contains unintended malicious behavior
- Any content that could cause direct harm if misused

Please **do not open a public Issue.**

Report it privately via one of the following:

- 📧 **Email:** mira3zzeldin@gmail.com
- 🔒 **GitHub Private Vulnerability Reporting :**
  Use the [Report a vulnerability](../../security/advisories/new) button in the Security tab of this repository.

---

## Response Timeline

| Action | Timeframe |
|---|---|
| Acknowledgement of report | Within 48 hours |
| Initial assessment | Within 5 business days |
| Resolution or disclosure decision | Within 14 days |

---

## Responsible Disclosure Policy

I am committed to **responsible disclosure**. If you report a valid security concern in good faith :

- I will acknowledge your report promptly
- I will work to resolve the issue as quickly as possible
- I will credit you in the resolution if you wish

Please allow reasonable time for resolution before any public disclosure.

---

## Out of Scope

The following are **not** considered security vulnerabilities for this repository :

- Theoretical attacks with no practical impact on this repo
- Content disagreements or factual errors in notes
  → Please use the [Content Feedback](.github/ISSUE_TEMPLATE) issue template instead
- Missing security hardening in example scripts used purely for education

---

## Legal

All content in this repository is intended strictly for **educational and ethical purposes.** Any unauthorized use of techniques documented here against systems without explicit permission is illegal and is not endorsed in any way.
